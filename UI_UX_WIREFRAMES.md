# Wallet Tracker - UI/UX Wireframe Descriptions

## Overview
This document provides detailed wireframe descriptions and user flow diagrams for all major screens in the Wallet Tracker application. These wireframes guide the UI/UX implementation.

---

## Design System Foundations

### Color Palette
```
Primary: #2C3E50 (Dark Blue-Grey)
Success/Income: #27AE60 (Green)
Alert/Expense: #E74C3C (Red)
Neutral: #ECF0F1 (Light Grey)
Text: #2C3E50
Subtle: #95A5A6 (Medium Grey)
```

### Typography
```
Headings: Sans-serif, Bold, 24-32px
Subheadings: Sans-serif, 16-20px
Body: Sans-serif, 14-16px
Numbers: Monospace, 16-18px
```

### Spacing
```
Base unit: 8px
Padding: 8px, 16px, 24px, 32px
Margins: 8px, 16px, 24px, 32px
Gap between elements: 16px, 24px
```

### Accessibility
```
Minimum color contrast: 4.5:1
Touch target minimum: 48x48px
Focus indicators: Visible 2-3px outline
```

### Key Elements

**Header**:
- Logo/title on left
- Current date
- Refresh button (icon)

**Current Balance Section**:
- Large, prominent number
- Last updated timestamp
- Subtitle: "Current Balance"

**Monthly Summary Cards** (3 columns):
- Income (green)
  - Large number with + symbol
  - Label: "Income"
- Expenses (red)
  - Large number with - symbol
  - Label: "Expenses"
- Net Flow (green/red depending on value)
  - Large number
  - Label: "Net Flow"

**Income Sources Section**:
- Heading: "Income Sources This Month"
- Simple table/list:
  - Source name | Amount
  - Scrollable if many sources
  - Shows only sources with activity

**Quick Actions**:
- 2-3 primary buttons:
  - "+ Add Entry"
  - "ğŸ“¤ Import Statement"
  - "ğŸ“Š View Analytics"
- Full-width or side-by-side

### Interaction Patterns
- Refresh button: Reload dashboard data
- Source name: Click to view source details
- Quick action buttons: Navigate to respective pages

### Responsive Design
- **Desktop**: 3-column card layout
- **Tablet**: 2-column card layout
- **Mobile**: 1-column card layout, stacked buttons

### Key Elements

**Amount Field**:
- Input type: number (decimal)
- Placeholder: "0.00"
- Currency symbol: $ on right side
- Validation: Must be > 0
- Error message if invalid

**Type Selector**:
- Toggle/Radio buttons: Expense | Income
- Default: Expense
- Shows/hides Source field based on selection

**Category Dropdown**:
- Multi-level categories
- Examples:
  - Income: Salary, Freelance, Investment, Bonus, Other
  - Expense: Groceries, Utilities, Transport, Entertainment, Other
- Allow custom entry

**Source Dropdown** (Income only):
- Only shows active income sources
- Required if type = Income
- Hidden if type = Expense

**Date Picker**:
- Default: Today
- Allow past/present dates
- Format: "Jan 30, 2026" or "01/30/2026"
- Cannot select future dates

**Notes Field**:
- Optional
- Placeholder: "Add any notes..."
- Max 500 characters

**Buttons**:
- Cancel: Gray, text-only or outline
- Save Entry: Green, filled button (primary)
- Success notification after save

### Form Validation
- Required fields marked with *
- Real-time validation feedback
- Submit button disabled until valid
- Clear error messages
- Red text or icon for errors

### Success Flow
1. User fills form
2. Clicks "Save Entry"
3. Success toast/notification appears
4. Form clears for next entry
5. Option: "Done" button to return to dashboard

### Key Elements

**Filter Section** (Collapsible):
- Date range: From & To date pickers
- Category: Multi-select dropdown
- Type: Income/Expense/All toggle
- Income Source: Dropdown (appears for income)
- Search: Free text search
- "Clear Filters" link/button
- Shows active filters below form

**Transaction List**:
- Chronological order (newest first)
- Columns:
  - Date (left, aligned)
  - Description (center, main text)
  - Category (right, subtle color)
  - Amount (right, bold, color-coded)
    - Expense: Red with - symbol
    - Income: Green with + symbol

**Transaction Row**:
- Click to view details
- Hover effect: Light background highlight
- Visual separation between rows
- Balance column (optional): Running balance

**Pagination**:
- Page numbers: [1] [2] [3]
- Previous/Next buttons
- "Showing X-Y of Z transactions"
- Configurable items per page (20, 50)

### Interactive Features
- Click row to expand details
- Filter combinations work together
- Real-time filter application
- Search as-you-type
- Sortable columns (by date, amount, etc.)

### Responsive Design
- **Desktop**: All columns visible
- **Tablet**: Hide balance column if needed
- **Mobile**: Stack horizontally, show description + amount prominently

### Key Elements

**Report Type Selector**:
- Dropdown with 4 report types:
  - Monthly Report
  - Yearly Summary
  - Category Breakdown
  - Income Source Summary

**Date Range**:
- From date picker
- To date picker
- Defaults: Start of month to end
- Cannot select future dates

**Format Selector**:
- Dropdown: PDF, CSV, Excel
- Default: PDF
- Shows file type and purpose

**Options** (Checkboxes):
- Include Charts (for PDF/Excel)
- Include Summary tables
- Include transaction details
- Options vary by format

**Buttons**:
- Preview Report: Shows sample
- Download Report: Generates and downloads file
- Download button disabled until valid selection

**Recent Downloads**:
- Shows last 5 exports
- File name, size, date
- Can re-download
